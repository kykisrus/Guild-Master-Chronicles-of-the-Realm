# Guild Master: Chronicles of the Realm

2D-менеджер гильдии с RPG-системой героев, строительством базы, процедурными заданиями, мировой картой, дипломатией, турнирами и визуальной автосимуляцией приключений.

Игрок управляет собственной гильдией, нанимает героев, формирует отряды, развивает гильд-холл, участвует в политике мира и наблюдает за тем, как герои проходят задания, данжи и турниры.

---

## Статус проекта

Проект находится в стадии полной перестройки интерфейса и визуального направления.

Текущий прототип переводится на единый Tiny Pixel Art-стиль.

Основной технический стек:

- **Godot 4**
- **GDScript**
- **2D**
- **PC: Windows / Linux**
- **Steam Deck**
- **Single-player**
- **Early Access**

---

## Основная концепция

Игрок выступает в роли Гильдмастера, обладающего редким даром.

Основная цель кампании:

- развивать гильдию;
- увеличивать славу;
- занимать верхние позиции в рейтингах;
- воспитывать героев высоких рангов;
- участвовать в ежегодной Битве гильдий;
- влиять на отношения с государствами;
- исследовать мир и данжи.

Кампания является бесконечной.

Формального окончания игры нет.

### Game Over

Игра заканчивается только в случае смерти Гильдмастера.

Преемников и воскрешения нет.

---

## Основной игровой цикл

```text
Нанять героя
→ сформировать отряд
→ выбрать задание
→ отправить отряд
→ наблюдать за перемещением по карте
→ завершать игровые дни
→ получить результат
→ развивать героев и гильдию
→ строить новые здания
→ открывать новые системы
```

Один игровой день равен одному тику.

День меняется только после нажатия кнопки **«Закончить день»**.

- 5 дней — игровая неделя;
- 90 дней — сезон;
- 360 дней — игровой год.

## Герои

Первая версия использует классы, уже представленные в Tiny Swords.

### Базовые классы

- **Воин**
- **Лучник**
- **Копейщик**
- **Монах**

### Персонал

- **Рабочий**

Другие классы будут добавляться в будущих обновлениях после появления подходящих совместимых спрайтов.

### Аватары

Большие индивидуальные портреты не используются.

Для каждого класса применяется маленький типовой аватар.

Различия героев формируются через:

- имя;
- уровень;
- ранг;
- характеристики;
- статус;
- историю;
- принадлежность к гильдии;
- цвет формы.

---

## Карточка героя

Карточка героя должна быть компактной.

Основные данные:

- типовой классовый аватар;
- имя;
- класс;
- уровень;
- ранг;
- здоровье;
- мораль;
- лояльность;
- усталость;
- статус;
- текущая задача.

Основные действия:

- добавить в отряд;
- открыть сведения;
- поговорить;
- отправить на тренировку.

---

## Ранги

Ранги доступны с первой версии.

```text
E → D → C → B → A → S → SS → SSS
```

Ранги влияют на:

- доступные задания;
- силу героя;
- статус в мире;
- стоимость найма;
- участие в турнирах;
- развитие класса;
- отношение других персонажей;
- рейтинг гильдии.

Повышение ранга является отдельным игровым событием.

---

## Отряды

Размер обычного отряда:

- минимум 2 героя;
- максимум 6 героев.

На турнирах:

- до 8 героев.

Гильдмастер не вступает в готовый отряд, но может:

- создать собственный отряд;
- отправиться на задание;
- участвовать в данже;
- присоединиться к турнирной команде.

Отряды имеют:

- название;
- состав;
- общую силу;
- сплочённость;
- состояние;
- текущую задачу;
- маршрут;
- оставшееся время.

---

## Задания

Задания генерируются процедурно.

Возможны комбинированные задания.

Карточка задания содержит:

- название;
- тип;
- регион;
- сложность;
- длительность;
- рекомендуемый состав;
- риск;
- основные награды.

Обычные задания разрешаются через автосимуляцию.

Игрок не управляет героями напрямую.

---

## Строительство

Строительство происходит:

- через свободные участки;
- через раздел строительства;
- через управляющего;
- через интерактивные объекты внутри помещений.

Строительство требует:

- золото;
- ресурсы;
- календарное время;
- выполненные зависимости;
- необходимый уровень гильдии.

Примеры зависимостей:

```text
Кузница → Оружейная
Таверна → Расширенный рекрутинг
Лазарет → Лечение раненых
Казарма → Тренировки
Зал совета → Дипломатия
```

## Дипломатия

Репутация рассчитывается отдельно для каждого государства.

Дипломатия включает:

- отношения;
- договоры;
- просьбы;
- региональные задания;
- политические события;
- войны;
- союзы;
- представителей государств;
- решения через зал совета.

Мир развивается независимо от игрока.

---

## Турниры и Битва гильдий

Турниры входят в первую версию.

Ежегодная Битва гильдий:

- объявляется заранее;
- участие необязательно;
- использует команды до 8 героев;
- влияет на рейтинг;
- влияет на мораль;
- влияет на славу;
- влияет на положение гильдии в мире.

Победа повышает мораль героев и положение гильдии.

Поражение уменьшает мораль и репутацию.

Guild_Master_Tiny_Roadmap.md

## Принципы разработки

- Каждый этап заканчивается рабочей сборкой.
- Сначала создаётся визуальный каркас.
- Затем добавляется взаимодействие.
- После этого подключаются игровые данные.
- Потом реализуется игровой результат.
- Полировка выполняется после готовности механики.
- Новые системы не должны перегружать игрока в начале.
- Интерфейс открывается постепенно.
- Все основные элементы должны работать на русском языке.
- Любая игровая система должна быть сохранена и восстановлена.
