# Визуальная концепция и структура интерфейса
## Guild Master: Chronicles of the Realm

**Версия:** 0.1  
**Движок:** Godot 4  
**Тип проекта:** 2D management RPG / guild simulator  
**Графический референс:** Esports Manager 2026  
**Цель документа:** зафиксировать визуальное направление, единый UI-kit и структуру ключевых экранов до начала полноценной отрисовки.

---

# 1. Общая визуальная идея

Интерфейс Guild Master: Chronicles of the Realm должен сочетать:

- структуру и читаемость спортивного менеджера;
- плотность информации без ощущения таблицы бухгалтерской системы;
- атмосферу фэнтези-гильдии;
- современный flat UI;
- тёмную базовую палитру;
- яркие акцентные цвета для статусов, редкости, предупреждений и действий;
- крупные портреты героев как главный эмоциональный элемент;
- лёгкую анимацию панелей, ресурсов, уведомлений и фона.

Главный принцип:

> Игрок должен ощущать, что управляет не меню, а живой гильдией, при этом вся информация должна быть доступна за 1–2 клика.

Референс Esports Manager 2026 используется прежде всего для:

- постоянной левой навигации;
- верхней панели с датой и кнопкой завершения дня;
- карточной подачи персонажей;
- экранов с большим количеством данных;
- сочетания таблиц, карточек и контекстных панелей;
- визуального разделения активных, будущих и завершённых событий;
- общей плотности и ритма интерфейса.

Прямое копирование оформления не требуется. Интерфейс должен быть адаптирован под собственную фэнтезийную стилистику.

## 2.2. Фэнтезийная стилизация

Современный интерфейс дополняется фэнтезийными деталями:

- тонкие металлические или каменные рамки;
- орнаменты только в заголовках и ключевых карточках;
- герб гильдии в верхнем левом углу;
- рунические элементы для магии;
- золотая окантовка для редких объектов;
- мягкие фоновые иллюстрации;
- затемнённые изображения гильд-холла, карты или интерьера;
- анимированные частицы в экранах магии и повышения ранга.

Важно не перегружать интерфейс пергаментом, свечами и декоративными рамками. Это менеджер, а не визуальная новелла.

## 3.2. Верхняя панель

Постоянный элемент на всех игровых экранах.

### Левая часть
- кнопка «Назад»;
- кнопка «Вперёд»;
- название текущего экрана;
- хлебные крошки при вложенной навигации.

### Правая часть
- портрет Гильдмастера;
- текущий день недели;
- дата;
- сезон;
- год;
- кнопка «Закончить день»;
- индикатор доступных срочных решений.

### Кнопка «Закончить день»
- всегда визуально заметна;
- зелёная при отсутствии критических решений;
- жёлтая, если есть предупреждения;
- красная, если есть блокирующее событие;
- перед завершением дня может показывать краткий список незавершённых действий.

---

## 3.3. Центральная область

Использует один из четырёх шаблонов:

### A. Карточный
Для героев, кандидатов, персонала, зданий.

### B. Табличный
Для финансов, трансферов, рейтингов, истории событий.

### C. Карта + параметры
Для тактик, карты мира, данжей.

### D. Календарь + список событий
Для квестов, турниров, строительства, тренировок.

# 4. UI-kit

## 4.1. Панели

Каждая панель имеет:

- фон `#1F2733`;
- скругление 6–10 px;
- тонкую границу;
- внутренние отступы 16–24 px;
- лёгкую тень;
- optional background illustration с opacity 5–12%.

---

## 4.2. Кнопки

### Primary
- зелёная или бирюзовая;
- основное действие;
- текст заглавными буквами;
- hover: яркость +10%;
- pressed: масштаб 0.98.

### Secondary
- тёмная;
- светлая рамка;
- используется для просмотра деталей.

### Danger
- красная;
- увольнение, отказ, отмена, рискованное действие.

### Disabled
- пониженная насыщенность;
- tooltip объясняет причину недоступности.

---

## 4.3. Карточка героя

Обязательные элементы:

- портрет;
- имя;
- ранг;
- уровень;
- класс;
- статус;
- мораль;
- физическое состояние;
- текущая активность;
- 1–3 ключевые особенности;
- кнопка подробностей;
- drag handle для перемещения в отряд.

### Состояния карточки
- доступен;
- на квесте;
- тренируется;
- ранен;
- отдыхает;
- участвует в турнире;
- недоступен;
- погиб.

---

## 4.4. Таблицы

- фиксированная строка заголовка;
- сортировка;
- фильтры;
- поиск;
- подсветка строки при наведении;
- контекстное действие справа;
- цветовые статусы;
- пагинация;
- возможность открыть карточку объекта по клику.

---

## 4.5. Статусы

Статусы должны быть оформлены короткими цветными плашками:

- Доступен;
- На задании;
- Ранен;
- Отдыхает;
- Обучается;
- Строится;
- Ожидает;
- Завершено;
- Провалено;
- Требует решения.

## Общий принцип реализации

- страницы не хранят глобальное игровое состояние;
- страницы получают данные через ViewModel / presenter;
- UI-компоненты переиспользуются;
- цвета и размеры задаются через Theme;
- данные страниц можно описывать JSON-конфигурацией;
- игровые данные хранятся отдельно от интерфейсных конфигураций.

# 9. Критерии готовности UI-концепции

Экран считается готовым к реализации, если определены:

- цель страницы;
- основной пользовательский сценарий;
- набор данных;
- обязательные действия;
- состояния загрузки;
- пустое состояние;
- состояние ошибки;
- подтверждения;
- адаптация под 16:9;
- поведение на Steam Deck;
- используемые компоненты;
- анимации;
- звуковая реакция;
- навигация назад.

# 10. Итоговое направление

Guild Master должен визуально ощущаться как современный глубокий менеджер, но не как клон киберспортивного симулятора.

Основная формула:

> Esports Manager по структуре + фэнтезийная гильдия по атмосфере + RPG по персонажам + Godot-friendly компонентная архитектура.
